# Security Policy

Please do not report security issues by opening a public GitHub issue.

If you believe you found a security problem, contact FreeProTool privately.

Do not include:

- Private files
- Passwords
- API keys
- Personal data
- Financial documents
- Medical documents
- Legal documents

## Public issues

Public GitHub issues are appropriate for:

- Broken tools
- Browser compatibility problems
- File processing bugs
- Documentation fixes
- Tool requests
- General privacy questions
